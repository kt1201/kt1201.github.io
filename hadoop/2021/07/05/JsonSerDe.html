서데(SerDe)

by kt1201

July 5, 2021
in Hadoop

서데(SerDe, Serializer/Deserialaizer)는 하이브가 데이터를 해석하는 방법을 제공한다. 예를 들어 하둡에 Json 파일을 따로 전처리하지 않고 서데를 이용하여 바로 집어 넣으면 하이브에서 Row 포맷으로 변환하여 읽을 수 있게 된다.

하이브는 서데와 파일 포맷을 이용하여 데이터를 읽고, 쓸 수 있다.

아래는 하이브가 서데를 이용하여 파일을 읽을 때와 쓸 때의 처리방법이다.

  • 파일 읽을 때 HDFS files -> InputFileFormat -> [key, value] -> Deserializer -> Row object
  • 파일 쓸 때 Row object -> Serializer -> [key, value] -> OutputFileFormat -> HDFS files

하이브 기본 서데

하이브에서 제공하는 기본 서데는 7가지(Avro, ORC, RegEx, Thrift, Parquet, CSV, JsonSerDe)가 있다. 각 서데는 STORED AS 에 지정하는 파일의 포맷에 따라 자동으로 선택된다.

가장 많이 쓰는 반정형 데이터 형식인 Json을 하둡에 넣고 서데를 이용하여 하이브로 조회해보자.

위는 json 데이터를 정렬시켜 놓았지만, hadoop에 실제 들어가는 json은 정렬하지 말고 한줄로 들어가야 한다.

json serde를 이용한 create문은 아래와 같다.

create table localdata (
	result struct<
		header: struct<
			columns: array<
				struct<
					rowNum: string,
					opnSfTeamCode: string,
					mgtNo: string,
					opnSvcId: string,
					updateGbn: string,
					updateDt: string,
					opnSvcNm: string,
					bplcNm: string,
					sitePostNo: string,
					siteWhlAddr: string,
					rdnPostNo: string,
					rdnWhlAddr: string,
					siteArea: string,
					apvPermYmd: string,
					apvCancelYmd: string,
					dcbYmd: string,
					clgStdt: string,
					clgEnddt: string,
					ropnYmd: string,
					trdStateGbn: string,
					trdStateNm: string,
					dtlStateGbn: string,
					dtlStateNm: string,
					x: string,
					y: string,
					lastModTs: string,
					uptaeNm: string,
					siteTel: string
				>
			>,
			paging: struct<
				pageIndex: string,
				totalCount: string,
				pageSize: string
			>,
			process: struct<
				code: string,
				message: string
			>
		>,
		body: struct<
		 	`rows`: array<
		 		struct<
		 			`row`: array<
		 				struct<
							rowNum: string,
							opnSfTeamCode: string,
							mgtNo: string,
							opnSvcId: string,
							updateGbn: string,
							updateDt: string,
							opnSvcNm: string,
							bplcNm: string,
							sitePostNo: string,
							siteWhlAddr: string,
							rdnPostNo: string,
							rdnWhlAddr: string,
							siteArea: string,
							apvPermYmd: string,
							apvCancelYmd: string,
							dcbYmd: string,
							clgStdt: string,
							clgEnddt: string,
							ropnYmd: string,
							trdStateGbn: string,
							trdStateNm: string,
							dtlStateGbn: string,
							dtlStateNm: string,
							x: string,
							y: string,
							lastModTs: string,
							uptaeNm: string,
							siteTel: string
						>
					>
		 		>
		 	>
		>
	>
)
ROW FORMAT SERDE 'org.apache.hive.hcatalog.data.JsonSerDe'
STORED AS TEXTFILE
LOCATION
  '/user/hive/st/localdata'
;

테이블을 생성했으면 hadoop에 데이터를 넣자.

$ hadoop fs -put localdata.json /user/hive/st/localdata

데이터에서 body만 조회를 해보자.

select
	msgarr.rowNum,
	msgarr.opnSfTeamCode,
	msgarr.mgtNo,
	msgarr.opnSvcId,
	msgarr.updateGbn,

	msgarr.opnSvcNm,
	msgarr.bplcNm,
	msgarr.sitePostNo,
	msgarr.siteWhlAddr,
	msgarr.rdnPostNo,
	msgarr.rdnWhlAddr,
	msgarr.siteArea,
	msgarr.apvPermYmd,
	msgarr.apvCancelYmd,
	msgarr.dcbYmd,
	msgarr.clgStdt,
	msgarr.clgEnddt,
	msgarr.ropnYmd,
	msgarr.trdStateGbn,
	msgarr.trdStateNm,
	msgarr.dtlStateGbn,
	msgarr.dtlStateNm,
	msgarr.x,
	msgarr.y,
	msgarr.lastModTs,
	msgarr.uptaeNm,
	msgarr.siteTel
from localdata ld lateral view explode(result.body.`rows`.`row`) msgtable as msgarr
;
